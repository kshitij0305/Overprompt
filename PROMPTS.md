# AI Prompt & Workflow Log

This document tracks prompts and AI-assisted workflows used during development.

The goal was not to generate the entire project automatically, but to accelerate:
- product thinking
- UX iteration
- recommendation logic refinement
- documentation structure
- MVP decision-making

---

# Product Direction

## Goal
Clarify what type of product the assignment should become.

### Prompt
“Does this assignment reward complex backend engineering or believable product thinking and presentation?”

### Outcome
Shifted development focus away from unnecessary backend complexity and toward:
- UX quality
- recommendation credibility
- SaaS-style presentation
- explainable product logic

This significantly changed the project direction.

---

# Landing Page Strategy

## Goal
Create a believable SaaS landing page structure.

### Prompt
“How should a lightweight SaaS audit landing page feel visually?”

### Outcome
Prioritized:
- clean typography
- structured spacing
- strong visual hierarchy
- minimal design language
- trust-oriented UI

Avoided excessive animations and overly flashy design patterns.

---

# Recommendation Engine Logic

## Goal
Avoid platform-biased audit recommendations.

### Prompt
“How can the audit engine focus on workflow fit and pricing efficiency instead of favoring specific AI platforms?”

### Outcome
Reframed the recommendation engine around:
- workflow alignment
- utilization efficiency
- spend-per-seat reasoning
- overlapping subscriptions
- collaboration plan justification

This made the recommendations feel significantly more believable.

Organize AI platform pricing research into readable documentation.

“How should pricing research be structured so it feels concise and reviewer-friendly?”

### Outcome

- pricing tables

- audit considerations
- verification dates
- source references

This improved readability and consistency across platforms.

# Architecture Decisions

## Goal
Keep the MVP architecture lightweight and explainable.

### Prompt
“How should the project architecture be structured for a fast-moving MVP?”

### Outcome
Chose:
- frontend-first architecture
- deterministic audit engine
- local React state
- modular component structure
- lightweight folder organization

Avoided premature backend complexity.

# UX & Product Tone

## Goal
Make the product feel trustworthy rather than gimmicky.

### Prompt
“What makes SaaS recommendation products feel credible?”

### Outcome
Focused on:
- transparent reasoning
- vendor-neutral recommendations

# Development Workflow

## Goal
Balance implementation speed with documentation quality.

### Prompt
“How should development and documentation be balanced during a short assignment timeline?”

### Outcome
Adopted a workflow where:
- product features were built during active development sessions
- documentation and reflections were updated incrementally afterward

This helped maintain:
- authentic development logs
- coherent project reasoning
- believable decision history

# Reflection

AI assistance was most useful for:
- structuring thinking
- refining product direction
- improving clarity
- challenging weak assumptions
- accelerating iteration

The most valuable part of the process was not automatic code generation, but using AI as a collaborative tool for:
- reasoning
- prioritization
- product refinement
- decision-making

This made the overall project feel significantly more focused and coherent.
